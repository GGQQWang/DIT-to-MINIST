# DiT-MNIST 条件扩散分类实验日志

## 1. 任务目标

本实验希望使用 DiT-B 构建 MNIST 手写数字条件扩散分类模型。

基本设定：

```txt
输入图像：MNIST 灰度图，尺寸 1x28x28
patch 大小：4x4
patch 数量：7x7 = 49
token 维度：DiT-B hidden_size = 768
类别数：10
```

数据流：

```txt
原始图像 [B, 1, 28, 28]
↓
Patch Embedding
↓
patch token [B, 49, 768]
↓
加入扩散噪声得到 x_t
↓
DiT 输入：x_t + timestep embedding + label embedding
↓
输出预测噪声或重构相关特征
```

分类方式不是直接加一个普通分类器，而是使用条件扩散重构误差做分类：

```txt
同一张测试图
分别假设 label = 0,1,2,...,9
↓
在每个 label 条件下做去噪 / 重构
↓
计算每个 label 的重构误差
↓
选择误差最小的 label 作为预测类别
```

也就是说，理想情况下：

```txt
正确类别条件下的重构误差应最小
错误类别条件下的重构误差应更大
```

## 2. 已完成实验

### 2.1 阶段一：仅条件去噪损失

最初的训练目标为：

```txt
loss = MSE(预测噪声, 真实噪声)
```

训练时输入：

```txt
带噪特征 x_t
扩散时间步 t
真实类别标签 y
```

模型需要预测加到 `x0` 上的真实噪声。

训练结果：

```txt
train_loss 明显下降
test_acc 长期只有 0.1~0.2 左右
```

20 epoch 后大致现象：

```txt
train_loss 从 0.339 降到 0.0067
test_acc 仍约 0.20
```

问题分析：

MNIST 的 `x_t` 本身仍包含较强的数字形状信息，模型即使不认真使用 `label`，也可以通过 `x_t` 和 `t` 学到通用去噪规律。因此训练 loss 会持续下降，但这不代表模型学会了类别条件。

这阶段模型更像是学到了：

```txt
给我一张带噪数字图，我尽量把噪声去掉
```

而不是：

```txt
给我一张带噪数字图和一个类别条件，我判断这个类别是否最能解释原图
```

结论：

```txt
仅靠条件 MSE 去噪损失，不能保证模型真正利用类别标签。
```

### 2.2 阶段二：随机负类对比损失

为了解决标签不起作用的问题，加入了随机负类对比。

训练时除了真实标签，还采样若干错误标签，目标是：

```txt
正确标签重构误差 < 错误标签重构误差
```

损失形式：

```txt
loss = denoise_loss + contrastive_weight * class_loss
```

其中 `class_loss` 由重构误差构造：

```txt
logits = -重构误差 / temperature
```

观察结果：

```txt
class loss 下降到约 1.5
test_acc 仍约 0.1
```

这个结果不能说明模型已经具备分类能力。原因是随机负类阶段比较的不是完整 10 类，而是：

```txt
1 个正确类别 + 若干随机错误类别
```

如果使用 3 个负类，则一共是 4 类，随机交叉熵基线为：

```txt
ln(4) = 1.386
```

如果使用 4 个负类，则一共是 5 类，随机交叉熵基线为：

```txt
ln(5) = 1.609
```

因此：

```txt
class loss ≈ 1.5
```

仍然接近对应类别数下的随机水平。

另外，这个训练目标和测试目标不完全一致：

```txt
训练：正确类别 vs 少量随机错误类别
测试：正确类别 vs 全部 9 个错误类别
```

所以即使随机负类上的损失下降，也不能保证真实类别会在完整 10 类比较中取得最小重构误差。

结论：

```txt
随机负类对比损失数值发生变化，但没有形成有效分类能力。
```

### 2.3 阶段三：全 10 类重构误差分类损失

为了让训练目标和测试目标一致，进一步改为全类别对比。

训练阶段对每个样本直接计算：

```txt
label=0 的重构误差
label=1 的重构误差
...
label=9 的重构误差
```

然后构造：

```txt
logits = -errors / temperature
class_loss = CE(logits, true_label)
```

这样训练目标变成：

```txt
真实类别的重构误差应该在 10 类中最小
```

观察结果：

```txt
denoise loss 开始下降
class loss 维持在约 2.3
test_acc 仍约 0.1
```

因为：

```txt
ln(10) = 2.302
```

所以 `class loss≈2.3` 表示仍接近 10 类随机水平。

问题分析：

虽然全 10 类训练目标更正确，但模型仍然优先走向通用去噪路径。也就是说，模型继续降低 `denoise_loss`，但不同类别条件下的重构误差没有被有效拉开。

结论：

```txt
全类别对比让目标更一致，但当前实现下分类信号仍然拉不动模型。
```

## 3. 当前核心问题

目前比较确定的问题不是训练轮数不够，而是训练目标和任务结构存在冲突。

核心现象：

```txt
去噪 loss 可以很低
分类 acc 仍接近随机
```

原因：

```txt
去噪任务本身太容易
x_t 已经携带大量原图信息
模型可以忽略 label 学通用去噪
重构误差之间的类别差距没有被稳定拉开
```

更直观地说：

```txt
模型学会了“怎样把带噪图修干净”
但没有学会“哪个类别条件最能解释这张图”
```

## 4. 关于阶段一 acc≈0.2 的解释

阶段一中 `test_acc` 曾达到约 `0.2`，而后续阶段更接近 `0.1`。这不一定说明阶段一更好。

阶段一模型主要是通用去噪器。在测试时，10 个类别条件下的重构误差可能非常接近，最终预测容易受以下因素影响：

```txt
不同类别 embedding 的随机偏置
训练中某些类别条件偶然形成的误差偏好
评估噪声采样带来的波动
MNIST 类别分布与模型误差偏置之间的偶然匹配
```

因此阶段一的 `0.2` 更可能是弱偏置或随机波动带来的结果，而不是稳定、可解释的分类能力。

真正有效的分类能力应当表现为：

```txt
正确类别重构误差系统性低于错误类别
class loss 明显低于对应随机基线
test_acc 持续稳定上升
```

## 5. 噪声强度与分类信号

后续讨论发现，扩散分类效果不仅取决于是否加入分类损失，还强烈依赖分类阶段使用的噪声强度。

在本任务中，噪声强度 `t` 相当于一个“判别难度旋钮”。

### 5.1 噪声太低

当 `t` 很小时：

```txt
x_t 非常接近 x0
原图结构几乎完整保留
```

模型可以直接根据 `x_t` 本身完成去噪，不需要认真使用类别标签。

例如原图是 `1`：

```txt
label=1 可以照着 x_t 去噪
label=2 也可能照着 x_t 去噪
label=7 也可能照着 x_t 去噪
```

结果是：

```txt
不同 label 下的重构误差差距很小
分类信号弱
```

### 5.2 噪声太高

当 `t` 很大时：

```txt
x_t 中原始数字结构大量丢失
模型难以判断原图到底是什么
```

此时类别标签可能主导重构结果。

例如原图本来是 `1`，但噪声已经很强，如果提示标签是 `2`，模型可能真的按 `2` 的条件去重构。

这会导致任务从“分类判别”退化成“条件生成”：

```txt
不是判断这张图更像哪个数字
而是给什么 label，就往什么数字方向生成
```

这不适合当前分类目标。

### 5.3 更合理的区域：中低噪声

更合理的噪声强度应处于中低噪声区域。

这个区域应满足：

```txt
原图整体结构仍然保留
局部判别笔画受到扰动
正确 label 能帮助恢复细节
错误 label 会与原图结构产生冲突
```

也就是说：

```txt
图像不能太干净
也不能太乱
```

太干净时，模型不需要 label。  
太乱时，模型可能只听 label。

理想状态是：

```txt
图像还看得出大概是什么
但关键局部存在不确定性
模型必须借助正确类别条件才能更好地还原
```

典型混淆区域包括：

```txt
1 和 7：顶部横线、倾斜笔画
3 和 8：闭合程度
4 和 9：上部闭合区域
5 和 6：下半部闭合区域
```

## 6. 关于“最优噪声区间”的前提

不能用完全未训练的模型直接寻找最优噪声区间。

因为要判断某个 `t` 是否有效，需要计算：

```txt
error_true = 真实标签条件下的重构误差
error_wrong_min = 错误标签中的最小重构误差
margin = error_wrong_min - error_true
```

这些指标依赖模型已经具备一定能力：

```txt
至少会基本去噪
最好对 label 有一定反应
```

如果模型还没训练，10 个 label 下的输出基本随机或相近，此时扫描 `t` 得到的 margin 没有可靠意义。

更合理的流程是：

```txt
阶段 A：先训练基础去噪模型
阶段 B：用已有模型固定参数扫描不同 t
阶段 C：根据扫描结果选择分类损失使用的噪声区间
阶段 D：在该区间强化分类训练
```

之前仅去噪训练 20 epoch 的 checkpoint 虽然分类能力弱，但已经具备基础去噪能力，可以作为噪声区间扫描的诊断模型。

需要注意：

```txt
如果所有 t 的 margin 都接近 0
说明主要问题不是 t 没选好
而是 label 条件本身没有被模型有效利用
```

## 7. 下一步实验路线

### 7.1 噪声时间步扫描

先用已有基础去噪模型做诊断，不训练，只评估不同时间步。

建议扫描：

```txt
t = 0, 10, 20, 50, 100, 150, 200, 300, 500
```

重点关注：

```txt
20~200
```

尤其是：

```txt
50, 100, 150
```

每个 `t` 记录：

```txt
test_acc
mean(error_true)
mean(error_wrong_min)
mean(margin)
median(margin)
margin < 0 的比例
```

其中：

```txt
margin < 0
```

表示至少有一个错误类别的重构误差小于真实类别，是分类失败样本。

理想现象：

```txt
mean(margin) > 0
median(margin) > 0
margin < 0 的比例下降
test_acc 上升
```

### 7.2 让训练和测试使用一致的有效噪声区间

如果扫描发现最佳区间集中在：

```txt
t = 50~150
```

则后续训练可以改为：

```txt
denoise_loss：仍使用全时间步，保持一般去噪能力
class_loss：只在 50~150 计算
eval_timesteps：使用 50,100,150
```

这样可以避免：

```txt
极低噪声：标签没有作用
极高噪声：模型按 label 条件生成
```

### 7.3 Hard Negative 训练

随机负类不一定是难分类样本。

例如真实数字是 `1`，随机负类可能采到：

```txt
0, 6, 8
```

这些类别和 `1` 差异很大，训练信号弱。

更有价值的负类可能是：

```txt
7, 2, 4
```

因此后续可以使用 hard negative：

```txt
先计算 10 个类别的重构误差
找出错误类别中误差最低的几个
重点惩罚这些最容易混淆的错误类别
```

可使用 margin ranking loss：

```txt
loss_wrong = max(0, margin_value + error_true - error_wrong)
```

含义：

```txt
如果错误类别误差已经明显大于正确类别，不再惩罚
如果错误类别误差接近或低于正确类别，强烈惩罚
```

这比单纯交叉熵更直接表达扩散分类需求：

```txt
正确标签应该最好地解释原图
错误标签不应同样好地解释原图
```

### 7.4 辅助分类头

如果重构误差分类信号仍然拉不动模型，可以考虑在 DiT 输出特征后增加辅助分类头：

```txt
DiT hidden features
↓
mean pooling
↓
Linear classifier
↓
10 类 logits
```

训练目标：

```txt
loss = denoise_loss
     + diffusion_class_loss
     + cls_weight * CE(class_logits, labels)
```

辅助分类头的作用不是替代扩散分类，而是强制 DiT 内部特征包含类别信息。

最终推理仍可以保留原来的扩散分类方式：

```txt
遍历 0-9
比较重构误差
选择误差最小类别
```

### 7.5 类别相关干扰

更进一步，可以考虑不只使用高斯噪声，还构造类别相关干扰。

普通扩散加噪：

```txt
x_t = sqrt(alpha_bar) * x0 + sqrt(1 - alpha_bar) * GaussianNoise
```

它的噪声不带类别结构。

但分类中真正困难的情况往往是“像另一个数字”的干扰：

```txt
1 被干扰得像 7
3 被干扰得像 8
5 被干扰得像 6
4 被干扰得像 9
```

可以考虑混淆类扰动：

```txt
x_mix = sqrt(alpha) * x_true + sqrt(1 - alpha) * x_confuser
```

其中 `x_confuser` 来自容易混淆类别的样本或特征。

目标是让模型学会：

```txt
即使局部像混淆类别，整体仍应按真实类别解释
```

这一步会偏离标准高斯扩散假设，更适合作为后续增强实验，而不是当前主线第一步。

## 8. 当前判断

当前最重要的判断是：

```txt
分类不是噪声越大越好
也不是噪声越小越好
```

更准确的目标是：

```txt
找到“原图信息仍在，但类别歧义被放大”的噪声强度
```

在这个强度下：

```txt
正确标签能帮助还原关键笔画
错误标签会与原图结构冲突
不同类别的重构误差才能真正拉开
```

下一步应优先做：

```txt
1. 使用已有基础去噪 checkpoint 扫描不同 t 的 margin
2. 根据扫描结果确定 class_loss 和 eval 使用的噪声区间
3. 再引入 hard negative 或辅助分类头
```
