# 第一阶段服务器运行命令

日期：2026年6月10日

本阶段环境只需要当前 `requirements.txt`：

```bash
pip install -r requirements.txt
```

## 1. 快速冒烟测试

先用每类少量样本确认代码、CUDA、数据下载正常：

```bash
python supervised_noisy_subspace.py \
  --device cuda \
  --train-per-class 200 \
  --eval-per-class 100 \
  --timesteps 0,100,200,300 \
  --dims 0,5,20,50 \
  --eval-noise-repeats 2 \
  --num-workers 4 \
  --output-dir ./几何假设验证/outputs_stage1_smoke
```

如果服务器没有 CUDA，把 `--device cuda` 改成：

```bash
--device cpu
```

## 2. 第一阶段主实验

使用完整 MNIST 训练集和测试集：

```bash
python supervised_noisy_subspace.py \
  --device cuda \
  --train-per-class 0 \
  --eval-per-class 0 \
  --timesteps 0,50,100,150,200,250,300,400,500 \
  --dims 0,1,2,5,10,20,50,100 \
  --train-noise-repeats 1 \
  --eval-noise-repeats 5 \
  --num-workers 8 \
  --output-dir ./几何假设验证/outputs_stage1_full
```

输出文件：

```txt
metrics_by_run.csv        每次测试噪声重复的结果
metrics_summary.csv       每个 t 和 d 的均值/方差汇总
explained_variance.csv    类别加噪子空间解释率
summary.md                自动生成的结果摘要
```

## 3. 加入子空间稳定性验证

该实验会用不同训练噪声种子构造两套类别子空间，并计算方向相似度：

```bash
python supervised_noisy_subspace.py \
  --device cuda \
  --train-per-class 0 \
  --eval-per-class 0 \
  --timesteps 0,50,100,150,200,250,300,400,500 \
  --dims 1,2,5,10,20,50,100 \
  --train-noise-repeats 1 \
  --eval-noise-repeats 3 \
  --stability \
  --num-workers 8 \
  --output-dir ./几何假设验证/outputs_stage1_stability
```

额外输出：

```txt
subspace_stability.csv
```

其中：

```txt
subspace_similarity 越接近 1，说明子空间方向越稳定。
```

## 4. 增强训练加噪样本

如果希望每个类别子空间由多个噪声版本估计，可以提高 `--train-noise-repeats`：

```bash
python supervised_noisy_subspace.py \
  --device cuda \
  --train-per-class 0 \
  --eval-per-class 0 \
  --timesteps 0,50,100,150,200,250,300,400,500 \
  --dims 0,1,2,5,10,20,50,100 \
  --train-noise-repeats 3 \
  --eval-noise-repeats 5 \
  --num-workers 8 \
  --output-dir ./几何假设验证/outputs_stage1_train_repeat3
```

## 5. 重点观察

重点看 `metrics_summary.csv` 和 `summary.md` 中的：

```txt
acc_mean
acc_std
margin_mean
```

如果实验支持假设，理想现象是：

```txt
t=0 不是最佳；
中等 t 的 acc_mean 更高；
中等 t 的 margin_mean 更大；
d 太小不足，d 太大可能引入噪声方向；
eval_noise_repeats 下 acc_std 较小；
subspace_similarity 在有效 t 和 d 下较高。
```
