## 2. 主实验：不同模型，同一中等噪声

先固定 `train_timestep=200`，比较模型学习原型吸引场的能力：

```bash
for model in linear mlp cnn dit-xs dit-s dit-b
do
  python attraction_field_models.py \
    --device cuda \
    --model $model \
    --epochs 20 \
    --batch-size 128 \
    --eval-batch-size 256 \
    --train-per-class 0 \
    --eval-per-class 0 \
    --train-timestep 200 \
    --eval-timesteps 200 \
    --noise-repeats 3 \
    --lr 1e-4 \
    --num-workers 8 \
    --output-dir ./吸引场验证/outputs_t200_${model}
done
```

重点比较：

```txt
eval_metrics.csv
train_log.csv
summary.md
```

## 3. 噪声强度消融

比较不同训练噪声强度：

```bash
for t in 0 100 200 300
do
  for model in linear mlp cnn dit-xs dit-s dit-b
  do
    python attraction_field_models.py \
      --device cuda \
      --model $model \
      --epochs 20 \
      --batch-size 128 \
      --eval-batch-size 256 \
      --train-per-class 0 \
      --eval-per-class 0 \
      --train-timestep $t \
      --eval-timesteps $t \
      --noise-repeats 3 \
      --lr 1e-4 \
      --num-workers 8 \
      --output-dir ./吸引场验证/outputs_t${t}_${model}
  done
done
```

判断重点：

```txt
哪些模型在 t=200 或 t=300 优于 t=0；
Linear 是否明显弱于非线性模型；
DiT-XS/DiT-S/DiT-B 是否随规模提升；
是否只有 DiT 呈现中等噪声最优。
```

## 4. 跨噪声泛化

固定训练噪声 `t=200`，评估多个 timestep：

```bash
for model in linear mlp cnn dit-xs dit-s dit-b
do
  python attraction_field_models.py \
    --device cuda \
    --model $model \
    --epochs 20 \
    --batch-size 128 \
    --eval-batch-size 256 \
    --train-per-class 0 \
    --eval-per-class 0 \
    --train-timestep 200 \
    --eval-timesteps 0,50,100,150,200,250,300,400,500 \
    --noise-repeats 3 \
    --lr 1e-4 \
    --num-workers 8 \
    --output-dir ./吸引场验证/outputs_generalize_t200_${model}
done
```

## 5. 输出文件

每次运行会生成：

```txt
train_log.csv       每个 epoch 的训练 loss
eval_metrics.csv    不同 eval_timestep 下的 acc、margin、距离、收缩比例
summary.md          最终摘要
checkpoint.pt       当前模型权重
```

## 6. 优先级建议

如果服务器时间有限，先跑：

```txt
linear
mlp
cnn
dit-xs
dit-b
```

并优先比较：

```txt
t=0
t=200
```

如果 `dit-xs` 已经接近 `dit-b`，可以减少后续 DiT-B 大规模实验。
